# Alcance del proyecto

## Idea general

`redux-travel-planner` será una aplicación de planificación de viajes para portfolio.

El objetivo principal es aprender y demostrar manejo de estado con Redux Toolkit en una app React más completa que una lista CRUD básica.

## Contexto del estudiante

Este proyecto parte de una base previa adquirida en el proyecto Mesa de Servicio.

El estudiante ya conoce de forma práctica:

- componentes;
- props;
- renderizado condicional;
- listas;
- formularios controlados;
- `useState`;
- `useReducer`;
- Context API;
- React Router;
- Tailwind CSS;
- consumo de APIs;
- Express;
- middlewares;
- autenticación;
- roles;
- Prisma;
- PostgreSQL.

Por eso, este proyecto no debe repetir explicaciones como si el estudiante estuviera iniciando desde cero.

Los conceptos no explorados en el proyecto anterior sí deben explicarse con más detalle, especialmente Redux Toolkit, RTK Query, React Hook Form, TypeScript gradual, mapas, popups, APIs externas y decisiones UI/UX más profesionales.

## Funcionalidades previstas

- Crear y administrar viajes.
- Agregar destinos a cada viaje.
- Buscar ciudades mediante API externa.
- Mostrar destinos en un mapa.
- Usar marcadores y popups para mostrar información rápida.
- Consultar clima del destino.
- Organizar actividades por día.
- Registrar presupuesto por categorías.
- Manejar autenticación simple.
- Consumir backend Express propio.

## Estilos

Los estilos se harán con Tailwind CSS.

Tailwind se usará desde las primeras fases para construir una UI más cuidada que la del proyecto anterior, con mejor jerarquía visual, responsive, estados vacíos, carga, errores y componentes reutilizables.

## Fuera del alcance inicial

- App empresarial.
- Pagos reales.
- Reservas reales.
- Integración con vuelos u hoteles reales.
- Chat en tiempo real.
- Permisos complejos por organización.
- Testing avanzado desde el inicio.
- TypeScript estricto en todo el proyecto desde la primera fase.

## Nivel esperado

El proyecto debe quedar en nivel intermedio guiado:

- más avanzado que un CRUD básico;
- más profesional en estado, formularios y UI/UX;
- suficientemente claro para portfolio;
- sin complejidad innecesaria.
